认知 00

全景地图:AI 擅长什么、不擅长什么

先建立一个诚实的能力边界认知,再谈提效。2026 年年中的通用大模型(Claude / GPT / Gemini)在下面这张表里的左列已经"稳定可用",右列则仍需人类兜底——把任务放错列,是一切失望的来源。

✅ 擅长(放心交给它)⚠️ 不擅长(必须人把关)
初稿生成:文章、代码、方案、邮件——从 0 到 60 分极快最后 20 分:观点、品味、取舍,仍是人的活
格式转换与结构化:乱笔记→表格、录音稿→纪要、需求→代码骨架事实精确性:会一本正经地编造(幻觉),重要事实必须核验
总结压缩:长文档、论文、会议记录提炼要点时效信息:训练截止后的事,不联网检索就不可信
翻译与润色:接近母语水平,风格可控长链路无验证任务:没有检查点的多步任务,错误会累积放大
头脑风暴:一次给 10 个方案,人来挑精确计算与计数:让它写代码算,别让它口算
跨领域入门讲解:任何学科的"耐心私教"责任与决策:医疗/法律/投资等高风险决定,AI 只能当参谋
代码:写脚本、修 bug、解释报错——有编译器和测试天然兜底"帮我做完全部":越模糊的全权委托,翻车率越高

三个反直觉的数据

  • 编程仍是第一大用途,但个人生活用途在猛涨。Anthropic 经济指数显示,与"计算机与数学"职业相关的任务约占 Claude 对话的 35%,居各类之首;同时个人用途占比从 35% 升至 42%(2025-11 → 2026-02)。
  • 过半任务是"协作"而非"自动"。57% 的任务是人机迭代协作,43% 是委托自动完成——说明当前最佳姿势仍是"AI 干活、人指挥和验收"。
  • 用错范式,AI 让你变慢。METR 2025 年随机对照试验:16 名资深开源开发者在自己熟悉的大型代码库上用 AI 工具,实测完成任务慢了 19%——但他们自我感觉快了 20%。
METR 研究的正确解读它证明的不是"AI 无用",而是:在"你本来就极熟"的领域,AI 增益最小;提效感受会严重高估实际提效。反过来推论——AI 增益最大的地方,恰是你不熟的领域、重复的体力活、和从零起步的初稿。选对场景 + 用对范式,才有真提效。本页后面全部内容,都在回答"怎么选、怎么用"。

认知 01

场景 × 收益矩阵:哪里提效最猛

八大场景按"提效潜力"排序。潜力大小不由技术炫酷程度决定,而由三个问题决定:有没有验证方法?错了代价多大?重复频率高不高?验证越容易、容错越高、重复越多的场景,越值得马上上 AI。

场景提效潜力为什么今天就能做的起步动作
编程 / 脚本★★★★★编译器、测试、报错信息构成天然验证闭环;初稿→可用的距离最短让 AI 写一个你手工做过 3 次以上的小脚本(如批量重命名文件)
写作 / 文档★★★★★初稿成本归零;结构、润色、多版本尝试全部提速;人只管观点与事实丢一堆素材给 AI,要它先出 3 版不同侧重的大纲
调研 / 决策辅助★★★★☆Deep Research 类功能 10-30 分钟产出带引用的调研报告,顶人肉半天选一个正纠结的购买/选型决策,跑一次深度调研并点开来源核验
学习新领域★★★★☆无限耐心的私教:任意深度追问、按你的水平解释、随时出题检验挑一个一直想学的主题,要求 AI"先摸底测试我,再定学习计划"
数据分析★★★★☆自然语言问表格已成熟;AI 写分析代码兼得灵活与可验证把一份 Excel/CSV 扔给对话模型,用中文问 3 个业务问题
自动化办公★★★☆☆收益极高但需先手动跑通流程;无验证的自动化是事故制造机把一个每周重复的流程写成清单,先让 AI 半自动跑一次
生活事务★★★☆☆行程规划、饮食方案、礼物点子、合同解读——低风险高频小决策下次做饭前:报上冰箱现有食材,要 3 个菜谱 + 采购补货清单
情感陪伴 / 自我梳理★★★☆☆HBR 百大用途榜 2025 年第一名;但注意它不是执业心理咨询师用"帮我把这团乱麻的想法整理成 3 个待决策问题"开始
别只盯着编程HBR 对全网真实用途的年度盘点显示,2025 年起排名前三的用途是心理疏导/陪伴、生活组织、寻找目标感——全是非技术场景。工程师圈子的声量让"AI = 写代码"成了错觉;对普通人,写作、调研、学习、生活侧的复利可能更大。

场景范式 02

编程场景最佳范式(Claude Code / Codex)

两家官方最佳实践 2026 年已高度趋同,可总结为一套四段式工作流 + 五个核心习惯。细节操作见本库 Claude Code 指南 与 Codex 指南,这里只讲"范式"。

官方共识:四段式工作流

  1. 探索(Explore)——进入计划模式(Claude Code 的 plan mode / Codex 的 /plan),让 AI 先读代码、回答问题,不动手改
  2. 计划(Plan)——要求产出具体实现计划:改哪些文件、什么顺序、怎么验证;人审计划比人审代码便宜 10 倍
  3. 实现(Implement)——退出计划模式执行,边写边跑测试;小任务(一句话能说清 diff 的)可跳过前两步直接干
  4. 提交(Commit)——审 diff、跑完整验证、写清楚提交信息;AI 的活也要过人这一关

五个核心习惯

习惯做法
给 AI 可运行的验证官方原话:给它一个能跑出 pass/fail 的检查(测试、构建、脚本、截图对比),闭环就自动转起来;没有验证,"看起来做完了"就是唯一信号,你就成了人肉验收员
说清完成标准Codex 官方推荐的任务结构:Goal(目标)+ Context(相关文件/文档)+ Constraints(约束)+ Done when(完成判据)——四要素齐了,返工率断崖式下降
小步迭代、尽早纠偏发现跑偏立刻 Esc 打断;同一个问题纠正两次还不对,别再纠了——/clear 重开,用学到的信息写个更好的初始提示
管好上下文上下文窗口是第一资源:无关任务间 /clear;大范围搜索交给子代理,别让主对话被文件内容灌满
沉淀项目记忆重复交代的规则写进 CLAUDE.md / AGENTS.md,并保持精简——判据是"删了这行 AI 就会犯错"才留
✗ 模糊委托"修一下登录的 bug"

AI 只能猜:哪个登录?什么症状?修到什么程度算好?
✓ 带验证的委托"用户反馈会话超时后登录失败。查 src/auth/ 里的 token 刷新逻辑。先写一个能复现问题的失败测试,再修复,最后跑整个 auth 测试套件确认全绿。"

非程序员也能上桌:vibe coding

用自然语言描述、让 AI 全程写码调试的"vibe coding"(Karpathy 2025 年 2 月造词,Collins 词典 2025 年度词),已让大量零基础用户做出了自用的记账 App、习惯打卡器、下料计算器等个人工具。社区统计称此类工具用户中过半从未写过代码(具体比例各家口径不一)。边界也要认清:自用小工具随便造;要对外收钱、存别人数据的产品,安全与运维仍需专业把关。

来源:Claude Code 官方 Best Practices · Codex 官方 Best Practices · Zapier:Vibe coding examples

场景范式 03

写作与文档场景范式

核心分工一句话:AI 出初稿与结构,人出观点与事实。把 AI 当"永不疲倦的初稿机 + 结构编辑",而不是代笔。

提示词四要素(MIT Sloan 教学共识)

有效写作提示 = 角色/背景(你是谁、写给谁)+ 任务与格式(产出什么、什么结构)+ 语气与受众 + 约束(字数、必须包含/避免的点)。四样都给,一次到位率最高;然后靠对话式迭代打磨:"更短""第二段展开""换成给老板看的语气"。

被验证的高收益用法

用法怎么做
三版大纲法丢素材,要 3 版不同侧重的大纲 → 人挑一版(或拼贴)→ 再逐节展开。比直接要全文的质量高得多
会议录音 → 纪要转写稿全文丢给 AI:"提炼决策、行动项(负责人+期限)、遗留问题三张清单"
长文压缩"压成 200 字摘要 + 10 条要点,保留所有数字和结论"——读文档前先读摘要,决定要不要精读
反向提纲把自己写的初稿丢回给 AI:"给这篇文章列反向提纲,指出逻辑断层和重复"——比"帮我润色"有效
去 AI 味给它 2-3 段你自己的旧文当风格样本;明令禁用陈词滥调("在当今快速发展的时代…");要求短句、具体名词、有数字
多语言润色中文思路 → 英文邮件/文档,注明场合与对象关系,产出接近母语水平
  1. 把原始素材(笔记/数据/链接)一次性全给 AI,先要大纲不要全文
  2. 人定大纲和核心观点——这是文章的灵魂,不外包
  3. 逐节生成初稿,每节给具体指令(要点、例子、长度)
  4. 人工核验所有事实、数字、引用——AI 的引用必须逐条点开确认
  5. 最后自己通读改一遍:改到"这话像你说的"为止
两条红线① 事实与引用必须人工核验——模型会自信地编造文献和数据,学术与商务场景尤其致命。② 别把敏感信息(客户数据、未公开业务数字、他人隐私)粘进公共 AI 服务,用企业版或本地方案。

来源:MIT Sloan:Effective Prompts for AI · 多方社区共识(做法均可在通用对话模型中直接复刻)

场景范式 04

调研与学习场景范式

调研:Deep Research 四步法

  1. 先写清问题边界——研究什么、不研究什么、判断标准、期望产出格式;边界越清,报告越有用
  2. 跑深度调研——ChatGPT / Gemini / Claude 的 Deep Research 类功能会自主检索、交叉阅读,10-30 分钟产出带引用的报告
  3. 追问与溯源——对关键结论逐条追问"这条的依据是哪个来源?",点开链接核验;引用造假是高发区
  4. 交叉验证——重要决策用第二个模型再跑一遍同题,对比分歧点;分歧处往往是需要人工深挖的地方

学习:把 AI 当私教,不当答案机

范式提示词模板
摸底定制"我想学 X。先出 5 道题摸底我的水平,再按结果给我定 4 周学习计划"
苏格拉底式"别直接给答案,用提问引导我自己推出来"——对抗"看懂了错觉"
费曼检验"我用自己的话讲一遍 X,你指出我哪里理解错了、哪里含糊"
难度调档"先讲给 12 岁孩子听,再讲专业版,再给我一个能上手的最小练习"
刻意练习"每天出 3 道递进难度的题,我答完你批改并讲解错因"
研究背书:会用是练出来的Anthropic「Learning curves」报告(2026-03)发现:熟手用户任务成功率比新手高约 10%,且熟手更多采用迭代协作而非一次性全权委托,更善于把模型能力和任务难度匹配。学习 AI 的最好方式就是高频使用 + 复盘哪种问法有效。

学术场景工具速览

文献发现:Elicit、Consensus、Semantic Scholar;论文精读与问答:NotebookLM、Claude(长文档);综述比对:Elicit 的跨研究提取表。原则不变:发现和摘要交给 AI,结论与批判留给自己。

场景范式 05

数据与自动化场景范式

数据分析:三个层级

层级做法适合
L1 自然语言问表格Excel/CSV 直接丢给对话模型(或 Julius 等专用工具),中文提问、要图表零基础;快速探索一份陌生数据
L2 AI 写分析代码让 AI 写 Python/SQL 脚本跑分析——代码可审可复跑,结果可验证,幻觉空间小要复用、要精确、数据大
L3 编码代理全流程Claude Code / Codex 直接读文件、清洗、分析、出报告,人验收结论多文件、多步骤的分析任务
为什么优先让 AI"写代码算"而不是"直接答"模型口算数字不可靠,但它写的代码是确定性执行的——这把"信 AI"换成了"信代码",而代码你可以审。这是数据场景最重要的一个观念转换。

自动化:五级阶梯,逐级上

  1. 手动对话——每次把重复任务丢给 AI 做,人盯全程
  2. 提示词模板——把跑通的问法存成模板/技能,一键复用
  3. 命令行非交互——claude -p "提示" / codex exec 接进脚本,批量处理
  4. 定时与钩子——每天自动跑:整理文件、汇总日报、盯监控(Codex 的 scheduled tasks / 系统 cron)
  5. 代理工作流——n8n / Zapier / 自建代理串起多系统;此时必须有失败告警与人工抽查
自动化两条铁律① 手动跑通三次才许自动化——流程本身还没稳定就自动化,等于把错误规模化。② 无验证不自动化——每一级自动化都要有可检查的产出(日志、diff、报表)和出错时的兜底(告警、人工审批点)。官方最佳实践对无人值守运行的要求同样是:先给检查,再谈放手。

  • 双开不冲突:Claude Code 与 Codex 各自独立认证、独立进程,常见组合是"日常用订阅内含的那个,难啃的重构换另一个";AGENTS.md 与 Agent Skills 是开放标准,项目知识沉淀在标准层,换代理零成本(Codex 还有 /import 直接吃 Claude Code 配置)。

3给 AI 可验证的闭环测试、样例、检查清单、对照样本——有 pass/fail 信号,AI 才能自我纠错;没有验证就别放手Claude Code Best Practices
4说清完成标准目标 + 上下文 + 约束 + "做到什么样算完"(Done when);模糊委托是返工之母Codex Best Practices
5小步迭代,两次纠错就重开首答只是起点;跑偏立刻打断;同一问题纠正两次无效,就带着教训清空重来官方最佳实践 + Learning curves(熟手更多迭代协作)
6人管品味与事实,AI 管体力观点、取舍、事实核验、最终责任永远在人;把 AI 输出当"初稿"而非"答案"METR / HBR / 多方共识
7把重复沉淀为资产跑通的问法存成模板,项目规则写进 CLAUDE.md / AGENTS.md,流程做成技能——个人提示词库是复利两家官方文档(Skills / 记忆文件)
一句话总纲把 AI 当成一个能力极强、毫无背景、过目就忘的新同事:给足上下文(1)、先对齐计划(2)、定好验收标准(3、4)、勤反馈(5)、关键决定自己拍板(6)、好的协作方式写进员工手册(7)。七条心法就是这一句话。
内功 08

真实实例集:每个场景一个可复刻模板

以下实例是对本页各范式的具体化,提示词可直接抄改。标注"实录"的来自公开报道,其余为范式复刻模板。

🐛编程 · 修一个 bugCODING

输入:"用户反馈:会话超时后重新登录会白屏。相关代码在 src/auth/,重点看 token 刷新。先写一个能复现的失败测试,再修复,跑通整个 auth 测试套件算完成。"

迭代:AI 提交计划 → 人确认"只动 auth,不重构其他" → AI 写测试(红)→ 修复(绿)→ 人审 diff。

产出:一个带回归测试的修复提交。关键在"失败测试先行"——这是验证闭环的最小实现。

📝写作 · 从素材到周报/文章WRITING

输入:本周的零散记录(聊天摘录、数据、待办完成情况)全部粘贴 +"写给不懂技术的老板看,300 字以内,三段式:成果 / 风险 / 下周计划。先给我 3 版大纲。"

迭代:挑大纲 → 逐段生成 → "第二段的风险太含糊,量化" → 人工核对所有数字。

产出:10 分钟出一份过去要 40 分钟的周报,且结构逐周稳定。

🔍调研 · 大额购买决策RESEARCH

输入:"预算 X 元买家用 NAS。需求:照片备份、两人用、静音优先、不折腾。用深度调研对比 2026 年在售的 3-5 款,给对比表 + 每款的差评集中点,附来源链接。"

迭代:报告出来后逐条追问差评来源并点开核验 → 换另一个模型跑同题 → 只深挖两份报告的分歧点。

产出:半小时得到过去两晚论坛爬楼的结论,且有据可查。

📊数据 · 月度报表分析DATA

输入:上传销售 CSV +"按月汇总各品类销售额,找出环比波动超 20% 的品类,分析可能原因,产出图表。用代码算,把代码一并给我。"

迭代:核对总数与原表一致(验证!)→ "把节假日因素考虑进去重跑" → 让 AI 把脚本参数化,下月直接复用。

产出:分析结论 + 一个可复跑的脚本——一次分析,月月复利。

⚙️自动化 · 下载文件夹自动归类AUTOMATION

输入:对 Claude Code:"写个脚本把 ~/Downloads 里的文件按类型归档到子文件夹,超过 30 天的截图移入回收站候选清单(不直接删)。先跑一次演习模式只打印计划,不动文件。"

迭代:看演习输出 → 修正误分类规则 → 真跑 → 手动确认三次没问题后,再挂定时任务。

产出:一个每周自动执行的整理任务。注意范式:演习模式 + 不做不可逆删除 + 跑通三次再自动化。

🎓学习 · 30 天入门一个新领域LEARNING

输入:"我想入门投资理财基础概念(不要荐股)。先出 5 道题摸底 → 按结果定 4 周计划,每周一个主题 → 每天 15 分钟:你讲一个概念 + 出 2 道题我答 + 批改讲解。"

迭代:每周末用费曼法:"我复述本周内容,你挑错" → 含糊处下周加练。

产出:一个贴合自己水平、天天有反馈的私教课程,成本为零。

🏠生活 · 一周晚餐 + 采购(实录风)LIFE

输入:"两大一小,孩子 5 岁;冰箱现有:(拍照或列清单)。排一周晚餐:30 分钟内能做、少油炸、周三要有鱼。产出:菜单表 + 按超市动线排序的采购清单。"

迭代:"周五那顿换掉,孩子不吃茄子" → 下周把"家庭忌口清单"存成固定模板开头。

产出:每周省下约一小时的决策消耗。非程序员做个人工具同理:公开报道里有零基础用户用 Claude + 八小时对话做出全家记账 App(见 Zapier 实例集,实录)。

上手 09

普通人学习路径:从零到熟练

研究结论先行:AI 熟练度是练出来的使用直觉,不是学出来的理论——熟手成功率高约 10%,差异主要来自"会不会迭代、会不会给上下文、会不会挑任务"。所以路径设计只有一个原则:永远围绕自己的真实任务练,不上抽象课。

🧗 开发者专属深化版如果你是移动端/客户端工程师想借 AI 转型全栈或做一人公司,本页方法论有一份场景特化的姊妹篇:Android 老兵转型全栈 · AI 时代路线图——12 周含验收的实战路线、Android→Web 认知映射与类比提问术。

四阶阶梯

  1. 第 1-2 周 · 建立手感:每天用对话模型解决 1 个真实小事(改邮件、解读体检单、做菜谱)。只练一件事:不满意就追问和补充信息,而不是放弃。目标:体感"首答只是起点"。
  2. 第 3-4 周 · 学会喂上下文:练习把背景材料一次给全(文件、截图、旧例子);对比"干问"和"带材料问"的质量差距。目标:内化心法 1 和 4(上下文 + 完成标准)。
  3. 第 2 个月 · 场景纵深:挑 2 个自己最高频的场景(如写作 + 调研),按本页 03/04 节的范式各跑 5 次完整流程;把跑通的问法存成个人模板库。目标:形成可复用资产。
  4. 第 3 个月起 · 上代理与自动化:装一个编码代理(Claude Code 或 Codex),从"写一个自动化小脚本"起步,按 05 节阶梯逐级上;开始用深度调研做大决策。目标:从"聊天"进化到"委托带验证的任务"。

常见误区对照表

误区纠正
期待一次成型,首答不满意就说"AI 不行"AI 是迭代式工具:追问、给反馈、补材料——熟手和新手的差距主要在这里
把 AI 当搜索引擎,只丢关键词给完整任务:背景 + 要求 + 完成标准;它是执行者不是索引
盲信输出,直接转发/提交事实、数字、引用必须核验;高风险内容(法律/医疗/财务)只当参考
外包思考:让 AI 替你决定观点与立场AI 出选项和论据,人做判断;长期外包思考会废掉判断力本身
囤教程、追新工具,不落地工具换得再勤,不如一个场景练 5 遍;先场景后工具
抽象学 AI(啃原理、背提示词大全)挑一个两周内要交付的真实产出,围绕它练;原理可以以后补
敏感信息随手粘贴个人隐私、客户数据、公司机密不进公共服务;用企业版或脱敏后再问
